Overview

This project was for CMPM 170 Game Design Studio I, the first course in three-part capstone for computer game design program.
Where students work in teams to design a substantial computer game with emphasis on novel, artistic concepts.
The prompt for this project was to create a RPG-style battle system focusing exclusively on mechanics and systems design.

Summary

WanQuest is a turn-based RPG that features three unique characters - a Mage, Thief, and Warrior - who must work together to defeat a powerful golem enemy.
Using classic RPG mechanics, players must strategically choose from a set of unique moves for each character and use items from their individual inventories to overcome the enemy's own set of powerful moves. The game's speed mechanic adds an element of risk to combat, forcing players to balance the risks of using slower, more powerful moves with the efficiency of using quick attacks. The battle ends when either all three characters die or if the golem's health pool is reduced to zero.
